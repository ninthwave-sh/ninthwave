# Status screen should show dependency trees visually

**Observed:** `ninthwave status` shows a flat list of items. When items have dependency chains (e.g., H-PRX-4 → H-PRX-5 → H-PRX-6 → H-PRX-7 → H-PRX-8 → M-PRX-9), the chain is not visually apparent.

**Impact:** Hard to understand at a glance which items are blocked by what, and how deep the dependency chain goes.

**Suggestion:** Show dependency trees with visual indentation/stacking in the status output. Items would be indented below their dependencies so the chain is clearly visible. Example:
```
✓ H-PRX-4     Merged    ...
  ✓ H-PRX-5   Merged    ...
    · H-PRX-6  Queued    ...
      · H-PRX-7 Queued   ...
```
